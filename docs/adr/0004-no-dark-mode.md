# ADR-0004: 无暗色模式设计原则

## 状态

Accepted

## 日期

2024-12-21

## 背景

在设计 UIX Design System 时，需要决定是否支持暗色模式（Dark Mode）。

### 行业趋势

大多数现代 UI 框架都支持暗色模式：
- Material Design 3：支持
- Apple Human Interface Guidelines：支持
- Ant Design：支持
- shadcn/ui：支持

### UIX Design 的独特定位

UIX Design 的核心设计理念是 **"白色画布"**：

> 纯净的白色背景象征着理性与清晰的思维空间，为 AI Agent 的交互提供最大的内容可读性和认知负载最低的视觉环境。

双主题色彩系统：
- **Rational Blue (#0284c7)**：代表理性、逻辑、技术
- **Sentient Gold (#f59e0b)**：代表感知、洞察、价值

这个设计语言建立在白色基底之上，暗色模式会破坏这个设计意图。

## 决策

**UIX 不提供暗色模式支持。**

理由：

1. **品牌一致性**
   - Lucid = 清晰、透明、理性
   - 白色基底是品牌核心视觉元素
   - 暗色模式会稀释品牌识别度

2. **设计复杂度**
   - 双模式意味着双倍的设计和测试工作
   - 颜色对比度、阴影、边框都需要两套方案
   - 资源有限时应聚焦

3. **目标场景**
   - AI Agent 应用主要是工作场景
   - 企业级应用通常在正常照明环境使用
   - 不是媒体消费类应用

4. **技术简化**
   - 不需要 CSS 变量切换逻辑
   - 不需要主题上下文管理
   - 组件实现更简单

## 后果

### 正面

- 设计和开发效率提升
- 品牌视觉统一
- 维护成本降低
- 测试矩阵简化

### 负面

- 部分用户偏好暗色模式
- 深夜使用场景体验下降
- 与系统暗色模式设置不同步

### 特殊场景处理

对于确实需要降低亮度的场景：
- 用户可使用操作系统/浏览器的亮度调节
- 代码块等特定组件可以使用深色背景（保持代码可读性）
- 这些是内容特性，不是主题切换

## 关闭 Issue

此决策意味着 GitHub Issue #6 (Dark Mode Support) 应被关闭，并引用此 ADR 作为说明。
